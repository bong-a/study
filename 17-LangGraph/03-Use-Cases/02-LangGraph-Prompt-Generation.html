요구사항을 물어보고 프롬프트를 만들어주는 챗봇

사용자에게 "어떤 프롬프트가 필요한지" 대화로 캐물어 요구사항을 모은 뒤, 그것을 바탕으로 완성된 시스템 프롬프트를 자동 생성하는 2단계 상태 머신입니다. 단계 전환 시점은 사람이 아니라 LLM이 스스로 판단합니다.

한눈에 보기

이 챗봇은 메타 프롬프트 생성기(Meta-Prompt Generator)입니다. "프롬프트를 대신 써주는 프롬프트"라고 생각하면 됩니다.

동작은 두 국면으로 나뉩니다 — ① 필요한 정보가 다 모일 때까지 사용자에게 질문하는 info(정보 수집) 단계, ② 모은 정보로 실제 프롬프트를 뽑아내는 prompt(생성) 단계. 정보가 충분히 모였다고 판단되는 순간 LLM이 도구(tool)를 호출하고, 그게 다음 단계로 넘어가는 스위치가 됩니다.

01전체 흐름

START → info로 시작해서, info 노드 뒤에 붙은 조건부 분기(get_state)가 세 갈래로 흐름을 결정합니다.

add_tool_message
"Prompt generated!"
ToolMessage 추가

prompt_gen_chain
META_PROMPT 적용

도구 호출됨

02두 개의 국면

시스템 메시지도, 입력으로 넣는 대화 기록도 단계마다 다릅니다.

Phase 1
정보 수집 (info)
  • 사용자에게 필요한 4가지 정보를 캐묻는다.
  • 정보가 부족하면 추측하지 말고 되묻는다.
  • 대화는 한국어, 결과 프롬프트는 영어로.
  • 정보가 다 모이면 PromptInstructions 도구를 호출한다.
get_messages_info() + template
Phase 2
프롬프트 생성 (prompt)
  • 도구 호출로 넘어온 요구사항을 재료로 삼는다.
  • META_PROMPT(OpenAI 가이드 기반)에 끼워 넣는다.
  • 도구 호출 이전 메시지는 전부 걸러낸다.
  • 완성된 시스템 프롬프트를 출력한다.
get_prompt_messages() + META_PROMPT

03수집하는 4가지 정보

LLM은 대화로 아래 4개를 다 파악하면, 이 구조(PromptInstructions)에 맞춰 도구를 호출합니다. 이게 "정보 수집 완료" 신호입니다.

objective str
이 프롬프트로 이루려는 목표가 무엇인가
variables List[str]
프롬프트 템플릿에 넣을 변수 목록
constraints List[str]
출력이 하면 안 되는 제약 조건
requirements List[str]
출력이 반드시 지켜야 할 요구사항

04분기의 핵심 — get_state

이 그래프에서 가장 헷갈리는 부분. info 노드가 끝난 뒤 마지막 메시지의 종류만 보고 다음 목적지를 정합니다. 위에서부터 순서대로 검사합니다.

마지막 메시지 상태이동의미
AIMessage + tool_calls 있음 add_tool_message 정보를 다 모아 도구를 호출함 → 생성 단계로 진행
HumanMessage 아님
(= 도구 없는 AI 답변)		 END AI가 되물었으니, 사용자 답변을 기다리며 턴 종료
그 외 (HumanMessage) info 아직 수집 단계 유지 — 정보 계속 모음
get_state()
def get_state(state):
    messages = state["messages"]
    # ① 도구 호출이 붙어 있으면 → 생성 단계로
    if isinstance(messages[-1], AIMessage) and messages[-1].tool_calls:
        return "add_tool_message"
    # ② AI가 되물은 상태면 → 사용자 답 기다리며 종료
    elif not isinstance(messages[-1], HumanMessage):
        return END
    # ③ 그 외 → 정보 수집 계속
    return "info"

💡 실제 대화 루프는 END → 사용자가 답변 입력 → 그래프 재호출 → info 순으로 돌아갑니다. MemorySaver가 thread_id별로 대화 기록을 이어주기 때문에, 종료됐다 다시 불려도 맥락이 유지됩니다.

05노드·함수 구성

이름역할
info_chain노드 시스템 템플릿 + 대화를 묶어 도구 바인딩된 LLM 호출
add_tool_message노드 "Prompt generated!" ToolMessage를 붙여 도구 호출을 마무리
prompt_gen_chain노드 META_PROMPT에 요구사항을 넣어 최종 프롬프트 생성
get_state분기 마지막 메시지로 다음 목적지 결정 (조건부 엣지)
get_prompt_messages도구 호출 이전 메시지를 걸러내는 필터 함수
MemorySaverthread_id별 대화 기록 저장 → 멀티턴 유지
그래프 조립
workflow.add_node("info", info_chain)
workflow.add_node("prompt", prompt_gen_chain)
workflow.add_node(add_tool_message)

workflow.add_edge(START, "info")
workflow.add_conditional_edges("info", get_state,
                               ["add_tool_message", "info", END])
workflow.add_edge("add_tool_message", "prompt")
workflow.add_edge("prompt", END)

graph = workflow.compile(checkpointer=memory)   # 대화 기록 저장

06실행 방식

while 루프로 사용자 입력을 계속 받아 그래프에 흘려보냅니다. q/Q를 입력하면 종료됩니다.

대화 루프
config = {"configurable": {"thread_id": str(uuid.uuid4())}}

while True:
    user = input("User (q/Q to quit): ")
    if user in {"q", "Q"}:
        break
    stream_graph(graph,
                 {"messages": [HumanMessage(content=user)]},
                 config=config,
                 node_names=["info", "prompt"])

·핵심 요약

  • 정보 수집(info)과 프롬프트 생성(prompt), 두 단계가 하나의 그래프 안에서 전환된다.
  • 단계 전환의 스위치는 LLM의 도구 호출이다 — 정보가 충분하면 스스로 PromptInstructions를 호출한다.
  • get_state는 마지막 메시지의 종류만 보고 도구 처리 / 종료 / 수집 계속을 가른다.
  • MemorySaver + thread_id 덕분에, 되물으며 여러 턴을 오가도 대화 맥락이 이어진다.

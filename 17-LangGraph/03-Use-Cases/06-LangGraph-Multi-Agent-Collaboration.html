멀티 에이전트 협업 네트워크

하나의 에이전트가 모든 도구를 떠안는 대신, 리서처와 차트 생성기가 그래프 위에서 결과를 주고받으며 하나의 질문을 나눠서 풀어내는 구조를 뜯어봅니다.

01 · 왜 나누는가

도구가 늘어날수록, 에이전트 하나로는 버거워진다

노트북은 AutoGen 논문에서 영감을 받아, "분할 정복(divide & conquer)" 방식으로 멀티 에이전트 네트워크를 구성합니다. 한 에이전트가 검색과 차트 생성을 모두 맡는 대신, 역할을 쪼개 각자 잘하는 일에 집중시킵니다.

단일 에이전트

  • 다뤄야 할 도구가 늘수록 도구 선택·호출 로직이 복잡해짐
  • 한 번에 처리할 컨텍스트(정보량)가 커져 비효율적
  • 도구 간 역할 경계가 흐려져 프롬프트가 비대해짐

멀티 에이전트 네트워크

  • 각 에이전트는 자기 전문 영역(검색 / 시각화)에만 집중
  • 필요하면 결과를 옆 에이전트에게 넘기고 나머지를 위임
  • 중앙 감독자 없이, 그래프의 조건부 엣지가 흐름을 결정
02 · LangGraph 개념 정리

공식 문서가 정의하는 멀티 에이전트 아키텍처

LangGraph 공식 개념 문서는 멀티 에이전트를 쓰는 이유를 세 가지로 정리합니다 — 모듈성(에이전트를 독립적으로 나누어 테스트·유지보수), 전문화(도메인 전문 에이전트가 전체 성능을 끌어올림), 제어(함수 호출보다 에이전트 간 통신 경로가 더 명확하게 드러남). 그 위에서 에이전트를 "어떻게 연결할지"를 다섯 가지 구조로 나눕니다.

아키텍처라우팅 결정 주체구조적 특징언제 쓰나
Network
이 노트북의 구조		 각 에이전트가 스스로 모든 에이전트가 서로를 호출할 수 있는 다대다(many-to-many) 연결. 정해진 리더가 없음 처리 순서나 계층이 뚜렷하지 않은 문제
Supervisor 감독자 LLM 노드 감독자가 매 턴 다음에 부를 에이전트를 결정하고 결과를 취합 중앙에서 흐름을 통제하고 싶을 때
Supervisor
(Tool-calling)		 감독자 LLM (도구 호출) 각 에이전트를 도구로 감싸고, 감독자가 ReAct 방식으로 함수처럼 호출 감독자 로직을 단순한 도구 호출로 줄이고 싶을 때
Hierarchical
Teams		 팀 감독자 + 최상위 감독자 팀 내부는 팀 감독자가, 팀 간에는 최상위 감독자가 라우팅하는 2단 구조 에이전트 수가 늘어 감독자 하나가 과부하될 때
Custom
Workflow		 고정 엣지 또는 Command 그래프 엣지로 순서를 못박거나(결정적), 도구 호출 결과로 부분 분기(동적) 흐름이 이미 정해져 있거나 예측 가능해야 할 때

이 노트북은 어디에 해당할까?

researcher와 chart_generator 사이에는 정해진 리더가 없고 둘 다 서로를 호출할 수 있다는 점에서 전형적인 2노드 Network 구조입니다. 다만 라우팅 방식은 공식 문서가 지금 보여주는 패턴과 조금 다릅니다 — 이 노트북은 Command(goto=...) 대신 add_conditional_edges() + 문자열을 반환하는 router() 함수로 같은 효과를 냅니다.

이 노트북 방식
def router(state):
    if "FINAL ANSWER" in last.content:
        return END
    return "continue"

workflow.add_conditional_edges(
    "researcher", router,
    {"continue": "chart_generator", END: END},
)
공식 문서의 Network 패턴Command
def researcher(state) -> Command[
    Literal["chart_generator", END]
]:
    response = model.invoke(...)
    return Command(
        goto=response["next"],
        update={"messages": [response["content"]]},
    )

Command는 "다음에 어디로 갈지"(goto)와 "상태를 어떻게 바꿀지"(update)를 한 번에 반환할 수 있어, 최근 문서는 조건부 엣지보다 이 방식을 더 권장합니다.

03 · 아키텍처

그래프 구조 한눈에 보기

앞서 본 Network 아키텍처를 조건부 엣지로 구현한 결과입니다. START에서 시작해 두 에이전트가 서로를 호출하며 루프를 돌다가, 누군가의 응답에 "FINAL ANSWER"가 등장하는 순간 END로 빠집니다. 이 판단은 매 턴 router() 함수가 내립니다.

researcher가 만드는 엣지
chart_generator가 만드는 엣지
고정 엣지 (START → researcher)

점선 = router()가 "FINAL ANSWER"를 감지했을 때의 종료 경로, 실선 곡선 = 아직 끝나지 않아 상대 에이전트로 넘기는 "continue" 경로입니다.

04 · 상태와 도구

에이전트들이 공유하는 것

두 에이전트는 MessagesState(메시지 리스트)를 공유합니다. 각자 결과를 만들면 name이 붙은 HumanMessage로 바꿔 상대가 "누가 무엇을 했는지" 알아볼 수 있게 합니다.

도구담당 에이전트역할실패 시 동작
TavilySearch
(tavily_tool)		 Research Agent 웹에서 최신 정보를 검색 (max_results=5) —
python_repl_tool Chart Generator Agent Python 코드를 실행해 차트를 생성하고 print 출력을 반환 예외를 잡아 "Failed to execute code. Error: ..." 문자열로 반환
상태 정의AgentState
class AgentState(TypedDict):
    messages: Annotated[
        Sequence[BaseMessage], operator.add
    ]  # Agent 간 공유하는 메시지 목록
    sender: Annotated[str, "The sender of the last message"]

공통 시스템 프롬프트 생성기make_system_prompt
def make_system_prompt(suffix: str) -> str:
    return (
        "You are a helpful AI assistant, collaborating with other assistants."
        " Use the provided tools to progress towards answering the question."
        " If you are unable to fully answer, that's OK, another assistant"
        " with different tools will help where you left off."
        " If you or any of the other assistants have the final answer or"
        " deliverable, prefix your response with FINAL ANSWER."
        f"\n{suffix}"
    )
06 · 라우팅 로직

다음 차례를 정하는 단 하나의 함수

감독 에이전트도, 별도의 오케스트레이터도 없습니다. 매 노드 실행 뒤 router()가 마지막 메시지만 들여다보고 계속할지 끝낼지 정합니다.

노드 실행 뒤 호출되는 조건 함수router
def router(state: MessagesState):
    messages = state["messages"]
    last_message = messages[-1]
    if "FINAL ANSWER" in last_message.content:
        # 누군가 작업이 끝났다고 선언 -> 그래프 종료
        return END
    return "continue"
노드 · 조건부 엣지 조립workflow
workflow = StateGraph(MessagesState)
workflow.add_node("researcher", research_node)
workflow.add_node("chart_generator", chart_node)

workflow.add_conditional_edges(
    "researcher", router, {"continue": "chart_generator", END: END}
)
workflow.add_conditional_edges(
    "chart_generator", router, {"continue": "researcher", END: END}
)

workflow.add_edge(START, "researcher")
app = workflow.compile(checkpointer=MemorySaver())

질문: "2010년 ~ 2024년까지의 대한민국의 1인당 GDP 추이를 그래프로 시각화 해주세요."

2

TavilySearch로 GDP 데이터 검색

검색 결과를 정리해 HumanMessage(name="researcher")로 변환 — 아직 "FINAL ANSWER"는 없음.

4
chart_generator

PythonREPL로 matplotlib 그래프 코드 실행

한글 폰트 설정(AppleGothic / 맑은 고딕)을 포함한 코드를 실행해 연도별 1인당 GDP 그래프를 그립니다.

"FINAL ANSWER" 감지 → END

에이전트
(Agent)

인공지능 시스템이 더욱 자율적이고 목표 지향적으로 작업을 수행하게 해주는 컴포넌트

Agent는 주어진 목표를 달성하기 위해 환경과 상호작용하며 의사 결정을 내리고 행동을 취하는 지능형 개체입니다.

Chain이 미리 정해진 순서로 실행된다면, Agent는 LLM을 추론 엔진으로 사용해 어떤 도구를 어떤 순서로 쓸지 스스로 결정합니다.
주요 특징

Agent의 5가지 특징

Agent를 일반 Chain과 구별 짓는 핵심 속성들입니다.

🧭

자율성

사전에 정의된 규칙이나 명시적인 프로그래밍 없이도 스스로 결정을 내리고 행동합니다.

🎯

목표 지향성

특정 목표나 작업을 달성하기 위해 설계되어 있으며, 목표 달성에 최적화된 행동을 선택합니다.

👀

환경 인식

주변 환경이나 상황을 인식하고, 변화하는 상황에 따라 전략을 적응시킬 수 있습니다.

🔧

도구 사용

검색, 계산, API 호출 등 다양한 도구를 활용하여 LLM 단독으로는 불가능한 작업을 수행합니다.

🔄

연속성

1회 수행이 아닌 반복 수행을 통해 목표 달성을 추구합니다. 목표가 달성될 때까지 루프를 계속합니다.

구성

LangChain Agent 구성요소

LangChain에서 에이전트는 4가지 핵심 요소가 서로 협력해 동작합니다.

🧠

Agent (에이전트 코어)

의사 결정을 담당하는 핵심 컴포넌트. LLM이 사용자 입력을 이해하고, 어떤 도구를 쓸지 추론하며, 최종 답변을 생성합니다.

create_tool_calling_agent
🔧

Tools (도구)

에이전트가 사용할 수 있는 기능들의 집합. 검색, 계산, 날씨 조회, 코드 실행 등 LLM 외부의 능력을 부여합니다.

@tool 데코레이터 · StructuredTool
🧰

Toolkits (툴킷)

관련된 도구들의 그룹. Gmail, GitHub, SQL 등 특정 서비스에 필요한 여러 도구를 묶어 패키지로 제공합니다.

GmailToolkit · SQLDatabaseToolkit
⚙️

AgentExecutor

에이전트의 실행을 관리하는 컴포넌트. Think→Act→Observe 사이클을 최종 답변이 나올 때까지 반복합니다. 루프 관리와 오류 처리를 담당합니다.

AgentExecutor(agent, tools, verbose)
작동 방식

에이전트의 6단계 작동 흐름

에이전트가 사용자의 요청을 받아 최종 답변을 생성하기까지의 과정입니다.

1

입력 수신

사용자로부터 작업이나 질문을 받습니다. 이것이 에이전트가 달성해야 할 목표가 됩니다.

2

계획 수립

주어진 작업을 완료하기 위한 단계별 계획을 세웁니다. LLM이 어떤 순서로 무엇을 할지 추론합니다.

3

도구 선택

각 단계에 적합한 도구를 선택합니다. 도구의 docstring을 읽고 어떤 도구가 목적에 맞는지 판단합니다.

4

실행

선택한 도구를 실제로 호출하여 작업을 수행합니다. API 호출, 검색, 계산 등이 이 단계에서 일어납니다.

5

결과 평가

수행 결과를 평가하고 목표 달성 여부를 확인합니다. 필요시 계획을 조정하거나 다른 도구를 선택하여 2~4단계를 반복합니다.

6

출력 생성

충분한 정보가 모이면 최종 결과나 답변을 사용자에게 제공합니다.

핵심 패턴

ReAct 루프

Reasoning + Acting. 에이전트 내부에서 Think → Act → Observe 사이클이 반복되는 구체적인 메커니즘입니다.

실제 실행 추적 예시 (verbose=True)
> Entering new AgentExecutor chain... Thought: 서울 날씨와 나눗셈 계산이 필요하다. 먼저 날씨 도구를 호출하겠다. Action: get_weather Input: "서울" Observe: 서울의 현재 날씨는 맑음, 23°C입니다. Thought: 날씨 정보를 얻었다. 이제 계산 도구를 사용하겠다. Action: calculate Input: "23 / 7" Observe: 3.2857142857142856 Thought: 두 정보를 모두 얻었다. 최종 답변을 생성할 수 있다. Final Answer: 서울은 현재 맑고 23°C입니다. 23 ÷ 7 ≈ 3.286입니다. > Finished chain.
구현

코드 예시

도구 정의 → 에이전트 생성 → 실행, 세 단계로 완성됩니다.

from langchain_core.tools import tool # @tool 데코레이터로 일반 함수를 에이전트 도구로 변환 # docstring이 LLM에게 도구 사용 방법을 알려주므로 정확히 작성해야 함 @tool def get_weather(city: str) -> str: """도시의 현재 날씨를 조회합니다. city에 도시명을 입력하세요.""" return f"{city}의 날씨: 맑음, 23°C" @tool def calculate(expression: str) -> str: """수학 표현식을 계산합니다. 예: '23 / 7', '100 * 0.05'""" return str(eval(expression)) # 툴킷(Toolkit)을 사용하는 경우 — 관련 도구들의 묶음 # from langchain_community.agent_toolkits import SQLDatabaseToolkit # toolkit = SQLDatabaseToolkit(db=db, llm=llm) # tools = toolkit.get_tools() tools = [get_weather, calculate]
from langchain.agents import create_tool_calling_agent, AgentExecutor from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate # 1. LLM — tool calling 지원 모델 필요 (GPT-4o, Claude 3 등) llm = ChatOpenAI(model="gpt-4o") # 2. 프롬프트 — {agent_scratchpad} 플레이스홀더 필수! # agent_scratchpad: Thought/Action/Observation 이력이 누적되는 공간 prompt = ChatPromptTemplate.from_messages([ ("system", "당신은 유용한 AI 어시스턴트입니다. 필요한 도구를 적극 활용하세요."), ("human", "{input}"), ("placeholder", "{agent_scratchpad}"), # ← 반드시 포함 ]) # 3. 에이전트 생성 — LLM + 도구 + 프롬프트 결합 agent = create_tool_calling_agent(llm, tools, prompt) # 4. AgentExecutor — 실행 루프 관리 agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True, # 실행 과정 출력 max_iterations=10, # 연속성: 목표 달성까지 최대 10회 반복 handle_parsing_errors=True # 오류 발생 시 재시도 )
# 단일 질문 — invoke result = agent_executor.invoke({ "input": "서울 날씨를 알려주고, 23을 7로 나눈 값도 계산해줘" }) print(result["output"]) # → 서울은 현재 맑고 23°C입니다. 23 ÷ 7 ≈ 3.286입니다. # 스트리밍 — 실시간으로 진행 상황 확인 for chunk in agent_executor.stream({ "input": "부산 날씨 알려줘" }): print(chunk) # 비동기 — FastAPI 등 async 환경 result = await agent_executor.ainvoke({ "input": "제주도 날씨는?" })
활용 사례

에이전트를 활용할 수 있는 분야

에이전트의 자율성과 도구 사용 능력이 빛을 발하는 실제 사용 시나리오들입니다.

🔍

정보 검색 및 분석

웹 검색, 데이터베이스 쿼리 등을 수행하며 여러 소스를 종합해 답변을 생성합니다.

⚡

작업 자동화

이메일 작성, 파일 처리, 데이터 변환 등 복잡한 워크플로우를 자동으로 처리합니다.

💬

고객 서비스

질문에 답변하고, 주문 조회, 환불 처리 등 다양한 고객 문제를 해결합니다.

📊

의사 결정 지원

데이터를 수집·분석하고 인사이트와 권장 사항을 제공해 의사결정을 돕습니다.

✍️

창의적 작업

글쓰기, 코드 생성, 이미지 생성 지시 등 창의적이고 복합적인 작업을 수행합니다.

평가

장점과 한계

에이전트를 도입하기 전 알아야 할 트레이드오프입니다.

장점
  • 복잡한 멀티스텝 작업의 자동화
  • 유연성과 적응성 — 예상치 못한 상황에도 대처
  • 다양한 도구 및 외부 API와의 통합
  • 반복 수행을 통한 점진적 목표 달성
한계
  • 제어와 예측 가능성의 어려움
  • 높은 계산 비용과 리소스 요구사항
  • 무한 루프나 잘못된 도구 선택 가능성
  • 디버깅이 Chain보다 복잡함
요약

핵심 정리

01

Agent는 LLM을 추론 엔진으로, Tools를 실행 수단으로 사용해 스스로 계획하고 목표를 달성합니다.

02

5가지 특징: 자율성 · 목표지향성 · 환경인식 · 도구사용 · 연속성이 Agent를 Chain과 구별 짓습니다.

03

내부 루프는 Think → Act → Observe. 최종 답변이 나올 때까지 반복하며 이를 ReAct 패턴이라 합니다.

04

프롬프트에 {agent_scratchpad}가 없으면 이전 추론 이력을 기억 못해 동작하지 않습니다.

05

verbose=True로 Thought/Action/Observation 추적, max_iterations로 무한 루프를 방지합니다.

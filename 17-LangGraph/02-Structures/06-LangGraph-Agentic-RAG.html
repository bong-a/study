Agentic RAG

LLM이 검색 도구 사용 여부를 스스로 판단하고, 검색 결과의 품질을 평가한 뒤 필요하면 질문을 재작성해 재시도하는 완전한 에이전트 RAG 구현.

아키텍처 — 그래프 상태 머신

AgentState

Tool
retriever_tool

PDF Retriever를 create_retriever_tool로 래핑. 에이전트가 도구 호출을 결정하면 LangGraph의 ToolNode가 실행한다.

        "<document><context>{page_content}</context>"
        "<metadata><source>{source}</source>"
        "<page>{page}</page></metadata></document>"
    ),
)
tools = [retriever_tool]

노드 구현

Agent State

state.py 
from typing import Annotated, Sequence, TypedDict
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages

class AgentState(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]

grade_documents — 조건부 라우팅 함수

grade_documents.py 
from typing import Literal
from pydantic import BaseModel, Field
from langchain_openai import ChatOpenAI

class grade(BaseModel):
    binary_score: str = Field(description="'yes' or 'no'")

def grade_documents(state) -> Literal["generate", "rewrite"]:
    model = ChatOpenAI(temperature=0, model=MODEL_NAME, streaming=True)
    llm_with_tool = model.with_structured_output(grade)

    prompt = PromptTemplate(
        template="""You are a grader assessing relevance of a retrieved document
to a user question. Retrieved document: {context}
User question: {question}
Give binary score 'yes' or 'no'.""",
        input_variables=["context", "question"],
    )
    chain = prompt | llm_with_tool

    messages  = state["messages"]
    question  = messages[0].content          # 첫 번째 = 원래 질문
    docs      = messages[-1].content          # 마지막 = 검색 결과
    scored    = chain.invoke({"question": question, "context": docs})

    if scored.binary_score == "yes":
        return "generate"
    return "rewrite"

agent · rewrite · generate

nodes.py 
def agent(state):
    # 도구가 바인딩된 LLM — tool call 여부를 스스로 결정
    model = ChatOpenAI(temperature=0, streaming=True, model=MODEL_NAME)
    model = model.bind_tools(tools)
    response = model.invoke(state["messages"])
    return {"messages": [response]}


def rewrite(state):
    # 검색 결과가 무관련할 때 — 질문의 의미를 개선하여 재시도
    question = state["messages"][0].content
    msg = [HumanMessage(content=f"""
Look at the input and try to reason about the underlying semantic intent.
Initial question: {question}
Formulate an improved question:""")]
    model = ChatOpenAI(temperature=0, model=MODEL_NAME, streaming=True)
    response = model.invoke(msg)
    return {"messages": [response]}


def generate(state):
    # 관련 문서 확인 후 최종 답변 생성
    question = state["messages"][0].content
    docs     = state["messages"][-1].content
    prompt   = hub.pull("teddynote/rag-prompt")
    llm      = ChatOpenAI(model_name=MODEL_NAME, temperature=0, streaming=True)
    rag_chain = prompt | llm | StrOutputParser()
    response  = rag_chain.invoke({"context": docs, "question": question})
    return {"messages": [response]}

그래프 구성 및 컴파일

graph.py 
from langgraph.graph import END, StateGraph, START
from langgraph.prebuilt import ToolNode, tools_condition
from langgraph.checkpoint.memory import MemorySaver

workflow = StateGraph(AgentState)

# 노드 등록
workflow.add_node("agent",    agent)
workflow.add_node("retrieve", ToolNode([retriever_tool]))
workflow.add_node("rewrite",  rewrite)
workflow.add_node("generate", generate)

# 엣지: START → agent
workflow.add_edge(START, "agent")

# 조건부 엣지: agent → retrieve | END
workflow.add_conditional_edges(
    "agent",
    tools_condition,                # tool call 여부 판단
    {"tools": "retrieve", END: END},
)

# 조건부 엣지: retrieve → generate | rewrite
workflow.add_conditional_edges("retrieve", grade_documents)

workflow.add_edge("generate", END)
workflow.add_edge("rewrite",  "agent")   # 재작성 후 재시도

graph = workflow.compile(checkpointer=MemorySaver())

Q1 "삼성전자가 개발한 생성형 AI 의 이름은?"
agent → retrieve → generate → END
문서에서 관련 내용 검색 성공 → DOCS RELEVANT → 답변 생성
Q2 "대한민국의 수도는?"
agent → END
LLM이 문서 검색 없이도 답할 수 있다고 판단 → 도구 호출 없이 즉시 종료
Q3 "테디노트의 랭체인 튜토리얼에 대해서 알려줘"
agent → retrieve → rewrite → agent → retrieve → rewrite → …
PDF에 존재하지 않는 주제 → DOCS NOT RELEVANT 반복 → rewrite 루프가 recursion_limit=10 초과시 GraphRecursionError 발생
recursion_guard.py 
from langgraph.errors import GraphRecursionError

try:
    stream_graph(graph, inputs, config, ["agent", "rewrite", "generate"])
except GraphRecursionError as e:
    print(f"GraphRecursionError: {e}")
재귀 한계 초과 — 문서에 없는 주제를 질문하면 rewrite → agent → retrieve → rewrite 루프가 무한 반복됩니다. 다음 튜토리얼에서 이를 해결하는 방법(최대 재시도 횟수 제한 등)을 다룹니다.

핵심 정리

LangGraph 패턴
tools_condition

prebuilt 헬퍼. 마지막 메시지가 AIMessage에 tool_calls를 포함하면 "tools", 없으면 END로 라우팅.

LangGraph 패턴
ToolNode

tool_calls 목록을 받아 도구를 실행하고 결과를 ToolMessage로 상태에 추가하는 prebuilt 노드.

평가 전략
Binary Grader

with_structured_output(grade)로 구조화된 yes/no 판단. 프롬프트 기반 채점으로 검색 품질을 제어.

루프 제어
recursion_limit

RunnableConfig에서 설정. 무한 루프 방지용 안전망. 초과 시 GraphRecursionError.
